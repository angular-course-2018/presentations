Coding pattern in which a class asks for dependencies from external sources rather than creating them itself.

Providing a service in Angular
game.service.ts
@Injectable()
export class GameService {
  cards: Card[];

  startGame() {...}
  endGame() {...}
  pickCard(card: Card) {...}
}

app.module.ts
@NgModule({
  providers: [ GameService ],
  //...
})
export class AppModule { }
Injecting a service in Angular
board.component.ts
@Component({/*...*/})
export class BoardComponent {
  constructor(
    private gameService: GameService,
  ){}

  letsGo() {
    this.gameService.startGame();
  }
}

Additional ways to provide services
From the service itself
game.service.ts
@Injectable({
  providedIn: 'root' | GameModule
})
export class GameService {
  cards: Card[];

  startGame() {...}
  endGame() {...}
  pickCard(card: Card) {...}
}

'root': available to all app via the root injector.
GameModule: available only when importing the GameModule in other module, same as the previous example.
Per component instance
board.component.ts
@Component({
  provide: [GameService],
  // ...
})
export class BoardComponent {
  // ...
}

note: since each provide creates an instance of the class, this will create instance per each component usage (instance).
